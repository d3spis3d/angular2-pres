Services

  • Handle most of the business logic of Angular2 apps
  • Communicate with the backend to load/store data
  • Much the same of AngularJS services
  • As I mention before Services are where almost of of our business logic lies in Angular2
  • Much the same as AngularJS they also handle the sending and receiving data from the server
  • Almost exactly the same as AngularJS services in design except they are classes

Services

        
            @Injectable
            export class HeroService {
                static get parameters() {
                    return [[BackendService]];
                }

                constructor(backendService) {
                    this._backend = backendService;
                }

                getHeroes() {
                    return this._backend.get('/api/v1/heroes');
                }
            }
  • Here is an example of a Service, notice that it has the Injectable decorator to help Angular2's dependency system
