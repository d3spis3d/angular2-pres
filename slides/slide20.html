Routing

  • Angular2 provides a router implementation
  • Like AngularJS, Angular2 provides a routing for SPA built with it and this router seems a bit more flexible than the ngRoute provided with AngularJS

Routing

        
            @Component({ ... })
            @RouteConfig([
              {path:'/crisis-center', name: 'CrisisCenter', component: CrisisListComponent},
              {path:'/heroes',        name: 'Heroes',       component: HeroListComponent},
              {path:'/hero/:id',      name: 'HeroDetail',   component: HeroDetailComponent}
            ])
            export class AppComponent { }
  • At the top level of the application there will be this top level AppComponent
  • It is here that the router is configured with this RouteConfig decorator
  • Each route has a url path, name and Component to be rendered

Routing

        
            @Component({
                template: `
                  
Component Router

                  

                    <a [routerLink]="['CrisisCenter']">Crisis Center
                    <a [routerLink]="['Heroes']">Heroes
                  

                  
                `,
            })
            @RouteConfig([ ... ])
            export class AppComponent { }
  • Looking at the AppComponents template we can see the router-outlet directive. When a route is matched the corresponding component will be rendered into this element
  • Angular2 also provides the routerLink directive which will be replaced with a link to the corresponding route
  • I'm a bit unsure at the moment how well this approach is going to scale for large apps with complex routing configuration?
