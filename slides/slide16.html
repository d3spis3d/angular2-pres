Data binding syntax

  • One way into child components / DOM
  • One way out of child components / DOM
  • Two way
  • Before I mentioned changes to the way data binding works compared to AngularJS
  • In Angular2 we have 3 kinds of data binding, and each has a different syntax and purpose

One way into

        
            <hero-detail *ngIf="selectedHero" [hero]="selectedHero"></hero-detail>
  • Here we have One way data binding into a child component
  • The square braces denote that data is passing from the component to a child component or DOM element
  • Here the selectedHero variable of our HeroesComponent is being passed into the hero variable of our HeroDetailComponent

One way out

        
            <div *ngFor="#hero of heroes" (click)="selectHero(hero)">
  • Here were have One way data binding out of a DOM element into the component who's template this is
  • The parentheses denote data being passed from a child component or DOM element to the parent component
  • In this example the one way data flow is from a click handler on the div in to the HeroesComponent

Two way
  • Lastly we have two way data binding which we will be familiar with from AngularJS
  • With two way data binding changes in the DOM element or Component will be reflected in the other as well
  • In this example the selectedHeroes name attribute is bound to the value of the input element
  • Two way data binding is denoted by this parentheses in braces syntax, which Angular2 recommends you remember by thinking of it as bananas in a box
