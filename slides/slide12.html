Structural directives

Built in Angular2 directives that modify the structure of templates

  • Angular2 comes with some built in directives for modifying the structure of components templates
  • These are generally similar to AngularJS built in directives but with different syntax

Structural directives

        
            

                <div *ngFor="#value in values">{{value}}</div>
  • ngFor is a replacement for the ngRepeat directive of AngularJS
  • The leading asterix is required syntax and the # symbol marks that we are declaring the local variable value in the template
  • ngFor with create as many copies of the div for each of the values in the values variable passing in from the component

Structural directives

        
            

                <div *ngIf="loading">Loading...</div>
  • ngIf is almost the same as AngularJS equivalent except the for the new syntax

Structural directives

        
            

                <div [ngSwitch]="value">
                    <p *ngSwitchWhen="true">Happy</p>
                    <p *ngSwitchWhen="false">Sad</p>
                    <p *ngSwitchDefault>Unknown</p>
  • ngSwitch is a new directive introduced for the case where we normally have a bunch of ngIf's where only one is active at a time
  • You will notice the [ brace notation which I will get to in a moment
  • Here depending on the what the value variable passed from the component is we will show one of these three p elements
